ARCHITECTURE ET LOGIQUE MATHÉMATIQUE

1. Détection des Anomalies (IQR)

Le moteur utilise la méthode de Tukey basée sur l'Écart Interquartile (Interquartile Range - IQR).

$$IQR = Q_3 - Q_1$$ $$Bornes = [Q_1 - k \cdot IQR, Q_3 + k \cdot IQR]$$

Où $k$ est le seuil de sensibilité réglable (par défaut 1.5). Cette méthode est plus robuste que le Z-Score car elle n'est pas influencée par les valeurs extrêmes elles-mêmes.

2. Stratégies d'Imputation (NaN)

Le traitement des données manquantes suit trois logiques au choix :

  • Médiane : Utilisée pour les distributions asymétriques (ex: prix immobiliers).
  • Moyenne : Adaptée aux distributions gaussiennes.
  • Zéro-Fill : Pour les variables de comptage optionnelles.

3. Pipeline ETL Vectorisé

Pour garantir la rapidité, le nettoyage n'utilise pas de boucles Python classiques mais la vectorisation NumPy/Pandas.

Complexité temporelle : $O(n \cdot m)$ où $n$ est le nombre de lignes et $m$ le nombre de colonnes.

4. Persistance des Données

Chaque session est archivée dans une base de données SQLite3. Cela permet une traçabilité complète des opérations de nettoyage effectuées sur les datasets.
